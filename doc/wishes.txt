 _____________________________________________________________________________
/                                                                             \
|   Lix                                                    Feature wishlist   |
\_____________________________________________________________________________/



Generell

    Automatisches Speichern von Replays
        Hier kann man mal drüber nachdenken.



Menue

    Weitere-Button
        Die letzte Seite nur teilweise zeigen, damit bei n Elementen, die
        auf dem Bildschirm Platz haben, immer Element Nr. pn am Anfang steht
        mit einer ganzen Zahl p.

    Scrollbalken statt Weitere-Button
        Selbsterklaerend.

    Navigation with cursor keys and return
        This might be useful for the big level browsers.

    Replays in the single player menu
        Levels with available replays should be marked differently in the
        single player menu, and maybe replays should be started directly
        from there.



Editor

    Automatisches Fortsetzen
        Das Terrain fuer eine Mehrspielerkarte nur einmal hinpflanzen und
        vom Spiel je nach Spielerzahl, die gerade dabei ist, vervielfaeltigen
        lassen.

    Rotate/mirror group
        If multiple elements are selected, rotating/mirroring should act on
        the objects' relative positions as well. The idea is to rotate/mirror
        a large part of the terrain to create symmectic levels more easily.



Gameplay

    Assignment shouldn't cancel pause
        Or should it? Has both advantages (flexibility) and disadvantages
        (extra keypress if you want to continue anyway).
        Possibly as an option.

    Show lixes in/out of hatch
        At the moment, you can only see the total of in-hatch and out lixes.
        One should be able to see the number of lixes in the hatch as well.

    Display hotkeys
        Display the skill hotkeys on top of the OSD buttons.

    Out of Skill
        Make a special sound when you're trying to switch to a skill you're
        out of (via hotkey or clicking on it)

    Changing priority/z-coordinate on selection
        If there's multiple pieces selected, their relative z-order should
        be maintained when changing terrain priority. Currently, they get
        messed up.

    Terrain selection rotation and mirroring
        Properly mirroring/rotating an entire selection in addition to the
        currently supported mode (i.e. affecting each piece separately).
        The old method is still useful to get shading of blocks right after
        flipping a whole selection.

    Non-permanent traps
        If lemmings of different players are under a trap, it prefers a
        certain type of lemmings (i.e. from a certain player). If non-
        permanent traps pick one lemming from each player currently in the
        trigger area, it would be balanced again

    Advanced trap definitions
        For traps, a lemming animation to be played as well as a soundeffect
        should be specifiable in the object definition. Lemming animation
        could be a custom one or one of the existing ones, and would allow
        for proper coloring of player lemmings, and maintaining the position
        of a lemming that gets into a trap.

    Roper
        Verschiesst ein Seil zum Drueberlaufen.

    Resolution
        Higher resolution than the internal 640 x 480 inside the game, even
        if it won't be used in the menu.

    Nuke disabling
        Each level could have the nuke for multiplayer disabled. Players
        can still activate a nuke-like function then, but it merely takes
        away control over their tribe and starts the overtime.
        
    End of time
        When the remaining time has reached zero, currently lems are still
        able to enter the exit as long as the nuke's exploder assignment
        was late on them. This allows level solve times over the maximum.
        Maybe lems should explode immedeately instead of exiting when walking
        into an exit with the clock being zero?



Netzwerk

    Observer mode
        People shall be able to watch a game, i.e. to pick an option besides
        the colors. Even better would be if a newcomer could view the ongoing
        game.

    Statistics
        After a game, you can view better statistics than now. The better
        screen could show how many lems every player got from whom.

    Level lokal speichern
        Netzwerklevel lokal zu speichern ermoeglichen, also ./data/netlevel.txt
        im Levels-Ordner unter beliebigem Namen speichern auf Kommando.

    Netzwerkpause
        Netzwerkpause einbauen, anstatt: Derzeit kann man auf [ESC] druecken,
        und das eigene Spiel pausiert. Es wird dann die Geschwindigkeit
        langsam wieder angeglichen, damit alle wieder gleichauf sind.

    Levelinfo
        Bei Mouseover in der Lobby über das Map-Image soll ein Tooltip-Fenster
        kommen und da drin steht dann, wie die Map heißt, wer der Autor ist,
        wie viel Zeit, Lemminge usw. usf.



Einzelspieler

    Highscore-Gewicht
        Benutzeroption einbauen, der bestimmt, ob das schnelle Schaffen von
        Levels wertvoller ist als das Sparen von Faehigkeiten.

    Highscore-Erweiterung
        Mehr als nur die Bestleistung eines Spielers im Einzelspielermodus
        speichern. Vergleichbarkeit mehrerer Benutzer untereinander einbauen.
